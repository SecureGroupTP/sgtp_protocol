# Хендшейк и Установка Сессии

## Базовый сценарий

1. Новый клиент отправляет `INTENT`.
2. Сервер ретранслирует этот `INTENT` существующим участникам комнаты.
3. Существующие участники отправляют новичку `PING`.
4. Новичок проверяет подпись/whitelist, считает DH secret, отвечает `PONG`.
5. Инициатор `PING` после `PONG` тоже считает DH secret.
6. После первого успешного обмена запускается `INFO`-фаза для получения полного списка участников.
7. Новый клиент допингивает недостающих peer-ов.
8. После завершения handshake запускается `CHAT_REQUEST`/выдача `CHAT_KEY`.

## Проверки безопасности

1. Подпись `PING/PONG` (ed25519).
2. Проверка `Client Hello`.
3. Проверка whitelist (по ed25519 pubkey).
4. Проверка timestamp окна.

## Выбор мастера

- Мастер определяется как peer с минимальным UUID.
- При уходе мастера роль автоматически переходит к следующему минимальному UUID.
