# История, FIN и KICK

## История (HSIR/HSI/HSR/HSRA)

1. Клиент отправляет `HSIR` broadcast.
2. Получает `HSI` с количеством сообщений от peer-ов.
3. Выбирает источник истории и отправляет `HSR(offset, limit)`.
4. Получает батчи `HSRA`.
5. `HSRA.message_count = 0` означает end-of-stream.

Особенность реализации: история пересобирается в MESSAGE-кадры и повторно подписывается.

## FIN

- Штатное завершение сессии peer-а.
- Получатель проверяет подпись и валидирует AEAD тег через текущий CK.
- Peer удаляется из локального состояния.

## KICK_REQUEST / KICKED

Нормативный сценарий протокола:
1. Участник видит timeout по ping и отправляет `KICK_REQUEST`.
2. Мастер сам пингует target.
3. Если нет `PONG` за `PING_TIMEOUT`, мастер рассылает `KICKED`.
4. После кика мастер ротирует CK.

Примечание по текущему Flutter-клиенту:
- Клиент обрабатывает входящий `KICKED`, но не использует `KICK_REQUEST` как активную runtime-ветку.
